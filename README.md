This project is a Login Page UI built as part of a software testing assignment. It demonstrates two types of testing:
Task 1 – Functional Test Cases: The login page was designed to validate four scenarios, a valid login that redirects to a dashboard, an invalid login that shows an error message, empty field submission that triggers a validation warning, and a basic SQL injection attempt that gets blocked and flagged.
Task 2 – Boundary Value Testing: The page enforces a password length rule of 8–12 characters, testing four boundary conditions below minimum (7 chars), at minimum (8 chars), at maximum (12 chars), and above maximum (13 chars).
The UI was built using HTML, CSS, and JavaScript
